Propõe-se o desenvolvimento de um sistema inteligente para monitoramento e gerenciamento de consumo utilizando algoritmos baseados em IA para conexão de dispositivos Bluetooth de áudio, alinhado com os pilares da Indústria 4.0. A solução será composta por um back-end robusto desenvolvido em Django, combinado com aplicativos mobile nativos: Kotlin para Android e Swift para iOS. A integração eficiente com dispositivos Bluetooth e APIs de fabricantes garantirá a coleta e processamento de dados em tempo real. O sistema contará com uma interface de usuário intuitiva e acessíveis, tornando assim de fácil uso, inclusivos e compreensíveis para uma ampla variedade de usuários, facilitando a navegação, visualização de métricas e a realização de configurações, promovendo uma experiência agradável e eficiente para o gerenciamento de dispositivos conectados.

4.3 Metodologia:

Este projeto possui características quanto a tamanho, legislação, normas, equipe, dentre outras características, que necessitam tanto de uma abordagem preditiva mais tradicional, quanto outras características que necessitam de abordagens mais ágeis. Assim, o Scrum será utilizado em conjunto ao estabelecido tradicionalmente no PMBOK, principalmente o desenvolvimento do software utilizará os ritos Scrum, enquanto outras partes como a pesquisa médica e o desenvolvimento de hardware terão seu método mais aproximado ao waterfall.

O Scrum é uma metodologia ágil para gerenciamento de projetos que se concentra em entregas iterativas e incrementais. A estrutura do Scrum inclui práticas, cerimônias e reuniões que as equipes realizam com regularidade. As etapas do Scrum que serão utilizadas neste projeto serão:
•	Sprint: É o período de tempo definido para o trabalho, geralmente entre uma a quatro semanas. Durante o Sprint, a equipe de desenvolvimento se concentra em entregar o que foi planejado. Para este projeto, ficou definido que cada ciclo terá a duração de quatro semanas, com entregas ocorrendo conforme a ordem de prioridade estabelecida no cronograma de atividades da equipe.
•	Reunião Diária: São reuniões curtas diárias, geralmente de 15 minutos, nas quais a equipe de desenvolvimento se reúne para discutir o progresso do trabalho e quaisquer impedimentos que possam estar enfrentando. Para este projeto, essas reuniões ocorrerão todos os dias no início do expediente, na primeira hora da manhã, para atualizar o status das atividades e identificar possíveis obstáculos.
•	Planejamento do Sprint: Após a contratação de toda a equipe de bolsistas, a coordenação do projeto pela UEA realizará uma reunião para planejar o Sprint. Nesta reunião, serão definidos os objetivos, as expectativas, as atividades a serem realizadas, as tarefas, as prioridades e os responsáveis por cada tarefa. Após essa reunião, o Cronograma de Atividades Técnicas será atualizado com os nomes dos responsáveis, permitindo que cada um insira o status das suas atividades. O objetivo é refinar todo o planejamento do projeto conforme o cronograma.
•	Reunião de Kick-Off: Realizada na fase inicial do projeto, tem o objetivo de alinhar todos os detalhes entre os stakeholders envolvidos — cliente, gestor do projeto e membros da equipe. Para este projeto, a reunião de kick-off contará com a participação de toda a equipe da empresa, incluindo a gestão do PD&I, Área Tecnológica e os bolsistas do projeto por parte da UEA. Nesta etapa, serão definidos o escopo, as metas, os prazos e os direcionamentos estratégicos.
•	Revisão do Sprint: Ocorre ao final do Sprint. Para este projeto, espera-se que a Salcomp, por meio de sua gestão técnica, se reúna com toda a equipe de desenvolvimento da UEA para revisar, de forma conjunta, o que foi entregue. Durante esta revisão, a equipe da UEA, representada pelo coordenador, demonstrará e celebrará os resultados do trabalho, verificará se o objetivo do Sprint foi atingido e se o resultado atendeu aos critérios de aceitação. A gestão técnica da Salcomp receberá o produto e realizará a homologação.
•	Retrospectiva do Sprint: É o evento que finaliza a Sprint ou ciclo de desenvolvimento do produto. Para este projeto, a equipe de desenvolvimento da UEA que participou da revisão do Sprint se reunirá para refletir sobre o ciclo anterior e identificar oportunidades de melhoria. Este será o momento de inspecionar o trabalho realizado, avaliar os pontos mencionados na reunião de revisão e identificar problemas e áreas que podem ser aprimoradas, criando um plano de ação para a próxima Sprint.

4.4 Linguagens de Programação, Ferramentas de Desenvolvimento ou Tecnológicas:

Python – Python é uma linguagem de programação amplamente reconhecida por sua simplicidade, versatilidade e eficiência. Sua sintaxe clara e direta facilita a leitura e escrita do código, o que aumenta a produtividade das equipes de desenvolvimento. Python conta com uma vasta gama de bibliotecas e framework, como o Flask e Django, que permitem o desenvolvimento de sistemas backend robustos, aplicações de inteligência artificial, processamento de imagens e APIs RESTful. Python também é conhecido por sua comunidade ativa, que fornece suporte contínuo e atualizações frequentes. Utilização no Projeto: Python será utilizado no desenvolvimento do backend do aplicativo mobile e em processos de análise de dados. Com o framework Django, será possível criar servidores escaláveis e seguros, gerenciar operações de banco de dados e disponibilizar APIs para integração com dispositivos Bluetooth. Python também será empregado em scripts para automação de processos, manipulação de dados e implementação de funcionalidades específicas, garantindo eficiência no processamento e flexibilidade no desenvolvimento.

Kotlin – Kotlin é uma linguagem de programação moderna e estaticamente tipada, projetada para o desenvolvimento de aplicações Android. Com interoperabilidade total com Java, Kotlin oferece recursos como inferência de tipo, null safety e programação funcional, facilitando a criação de aplicativos concisos e menos propensos a erros. Sua sintaxe expressiva e clara ajuda a reduzir a quantidade de código boilerplate, acelerando o desenvolvimento e melhorando a manutenção. Kotlin é oficialmente suportado pelo Google e faz parte do ecossistema Android. Utilização no Projeto: Kotlin será a linguagem principal para o desenvolvimento da versão Android do aplicativo. A escolha de Kotlin permitirá implementar funcionalidades específicas para gerenciamento de dispositivos Bluetooth com alto desempenho e segurança. Combinado com Jetpack Compose, será possível criar interfaces de usuário modernas, dinâmicas e altamente responsivas, proporcionando uma experiência de uso intuitiva e agradável. Kotlin também facilitará a integração com APIs externas e serviços nativos do Android, assegurando que o aplicativo utilize os recursos do sistema de forma eficiente.

Jetpack Compose – Jetpack Compose é o toolkit moderno do Android para a criação de interfaces de usuário declarativas e reativas. Ao simplificar o desenvolvimento de layouts, Jetpack Compose permite construir UIs de maneira modular e reutilizável, reduzindo a complexidade do código e facilitando a manutenção. Ele oferece uma abordagem declarativa para a criação de componentes visuais, o que torna a atualização e a modificação da interface muito mais rápidas e intuitivas. Além disso, Jetpack Compose é altamente compatível com outros componentes do ecossistema Android, garantindo uma integração eficiente e fluida. Utilização no Projeto: Jetpack Compose será empregado na criação da interface gráfica do aplicativo Android, permitindo desenvolver telas dinâmicas e adaptáveis às diferentes necessidades dos usuários. Com Jetpack Compose, a equipe poderá implementar rapidamente mudanças no design e nos componentes visuais, garantindo uma interface intuitiva e responsiva para o gerenciamento dos dispositivos Bluetooth. Isso assegurará que a experiência do usuário seja fluida, visualmente atraente e alinhada às melhores práticas de design moderno.

GitHub – GitHub é uma plataforma de desenvolvimento colaborativo que utiliza o sistema de controle de versão Git. Ele oferece um ambiente centralizado para armazenar, versionar e gerenciar o código-fonte de projetos, permitindo que equipes trabalhem de forma integrada e organizada. GitHub facilita o rastreamento de mudanças, a revisão de código, a resolução de conflitos e a colaboração assíncrona. Além disso, a plataforma fornece ferramentas para automação, integração contínua e implantação contínua (CI/CD), que ajudam a manter a qualidade e a eficiência do processo de desenvolvimento. Utilização no Projeto: GitHub será utilizado como repositório central para armazenar o código-fonte do aplicativo e dos componentes associados. A equipe de desenvolvimento utilizará GitHub para versionamento de código, gerenciamento de branches, rastreamento de problemas e colaboração em tempo real. Ferramentas como GitHub Actions serão aplicadas para automatizar os processos de build e teste, garantindo a integração contínua e a entrega rápida de novas funcionalidades. A plataforma também facilitará a documentação do projeto e a revisão de código, assegurando a qualidade e a consistência das entregas.

React Native – React Native é um framework de código aberto desenvolvido pelo Facebook que permite criar aplicativos móveis para Android e iOS usando JavaScript e React. Com React Native, os desenvolvedores podem escrever uma única base de código que é compilada para aplicações nativas, oferecendo desempenho e aparência equivalentes aos aplicativos desenvolvidos nativamente. React Native é altamente eficiente para projetos que precisam ser implementados rapidamente em múltiplas plataformas, reduzindo custos e prazos de desenvolvimento. Utilização no Projeto: React Native será utilizado para desenvolver a versão multiplataforma do aplicativo, garantindo que usuários de Android e iOS possam acessar as mesmas funcionalidades de gerenciamento de dispositivos Bluetooth. A capacidade de reutilização de componentes e a ampla gama de bibliotecas disponíveis acelerarão o desenvolvimento e simplificarão a manutenção do código. Com React Native, a equipe poderá implementar atualizações simultaneamente para ambas as plataformas, garantindo consistência na experiência do usuário.

Django – Django é um framework de alto nível para o desenvolvimento de aplicações web em Python. Ele é conhecido por sua segurança, escalabilidade e facilidade de uso. Django permite a criação rápida de aplicações robustas, fornecendo recursos integrados para autenticação, gerenciamento de banco de dados, roteamento de URLs e APIs RESTful. Com Django, é possível desenvolver soluções backend seguras e eficientes, garantindo a integridade dos dados e o desempenho das operações. Utilização no Projeto: Django será utilizado como o backend principal para o aplicativo mobile, gerenciando a lógica de negócios, o armazenamento de dados e a criação de APIs para integração com dispositivos Bluetooth. Django oferecerá uma estrutura segura e escalável para manipular dados de usuários e dispositivos, além de garantir a implementação de funcionalidades como autenticação e controle de acesso. A integração com bibliotecas de Python facilitará o processamento de dados e a análise de informações coletadas, garantindo que o backend ofereça suporte eficiente e confiável ao aplicativo mobile.

Machine Learning - Machine Learning (ML), ou aprendizado de máquina, é uma tecnologia central no projeto, permitindo que o sistema analise dados, reconheça padrões e tome decisões automaticamente. No contexto desta aplicação, o ML será utilizado para melhorar a experiência do usuário e otimizar o funcionamento dos dispositivos conectados. Por meio da análise de padrões de uso, o sistema será capaz de identificar as preferências individuais do usuário, como configurações frequentes de áudio ou hábitos relacionados à saúde, e oferecer recomendações personalizadas.
Além disso, o aprendizado de máquina permitirá a previsão de necessidades do usuário em tempo real, ajustando automaticamente configurações como equalização de áudio ou emissão de alertas relacionados a métricas obtidas nos sensores dos wearables. Outra funcionalidade importante é a detecção de anomalias, onde algoritmos de aprendizado não supervisionado identificarão comportamentos incomuns, como falhas em dispositivos ou métricas fora do padrão esperado, acionando ações corretivas de forma imediata.
Com o uso de frameworks avançados como TensorFlow ou PyTorch, o projeto empregará modelos supervisionados e não supervisionados para processar grandes volumes de dados de maneira eficiente e com baixa latência, garantindo um sistema robusto, adaptável e em constante evolução. O Machine Learning, assim, desempenha um papel essencial ao alinhar o projeto às demandas de personalização, automação e inteligência da Indústria 4.0.

Esse tipo de aplicativo combina funcionalidades de controle de áudio, monitoramento de treinos e coleta de dados de wearables, integrando-se também com serviços externos como Google Fit e Strava. A primeira etapa é o planejamento e definição do escopo. Nesse momento, é necessário identificar as necessidades do público-alvo e as funcionalidades principais do aplicativo. Para um app focado em wearables, isso inclui o pareamento com dispositivos, sincronização de dados, interface intuitiva para controle de áudio e visualização de métricas de sensores. Também é importante mapear quais serviços de terceiros serão integrados, como APIs de saúde e fitness.
Com o escopo definido, a fase de design começa. Aqui, é desenvolvido o protótipo do aplicativo, incluindo a interface do usuário (UI) e a experiência do usuário (UX). O objetivo é garantir que o app seja visualmente atraente, intuitivo e funcional. A interface deve permitir que os usuários configurem seus dispositivos, visualizem dados de sensores e controlem funcionalidades como reprodução de áudio de forma simples. Com base nos objetivos do projeto, através da utilização de software para design de interfaces chamado Figma, foi idealizado uma série de interfaces para apresentação desses dados de wearables e configuração dos dispositivos Lity e de outros parceiros, com a criação de um ambiente integrado para saúde.
Para alcançar os objetivos estabelecidos e garantir o sucesso do projeto, serão aplicadas as tecnologias apresentadas por meio das metodologias ágeis Scrum e Waterfall. Essa abordagem se dará pela divisão do cronograma em atividades específicas e pela organização da equipe em grupos de desenvolvimento, sendo eles: Android, IOS e Back -End.

O desenvolvimento técnico é dividido em várias camadas. Para aplicativos híbridos que funcionam tanto em Android quanto em iOS, frameworks como Flutter ou React Native podem ser usados para compartilhar grande parte do código. Esse desenvolvimento é dividido em três principais camadas:
1.	Camada de apresentação (Presentation): Envolve a criação da interface gráfica do usuário e a lógica de apresentação, como transições de telas e exibição de dados em tempo real. Nesta camada, é crucial garantir que os elementos visuais sejam consistentes entre as duas plataformas, respeitando suas diretrizes específicas (Material Design no Android e Human Interface Guidelines no iOS).
2.	Camada de negócios (Business): Aqui é implementada a lógica do aplicativo, como o fluxo de pareamento dos dispositivos wearables, a sincronização de dados e a integração com APIs de terceiros. Essa camada gerencia também a segurança do app, garantindo que os dados pessoais dos usuários sejam protegidos.
3.	Camada de dados (Data): Responsável por gerenciar o armazenamento e a recuperação de informações. Para um app como este, é necessário combinar o armazenamento local, usado para dados acessados com frequência, e o armazenamento remoto, que centraliza as informações no servidor. Além disso, é implementada a comunicação com dispositivos wearables via Bluetooth ou Wi-Fi e a integração com plataformas de saúde, como Google Fit e Strava.

O backend desempenha um papel fundamental, servindo como um ponto central para processamento de dados e integração com serviços externos. Ele é responsável por armazenar informações de saúde e treinos, realizar análises, gerenciar usuários e enviar notificações. APIs REST ou GraphQL são comumente utilizadas para conectar o backend ao aplicativo móvel.
Durante o teste e validação, o app é submetido a rigorosos testes em ambas as plataformas para garantir que funcione corretamente em uma ampla variedade de dispositivos. Isso inclui testes de desempenho, usabilidade, compatibilidade com diferentes wearables e segurança. Por fim, o aplicativo é lançado nas lojas (Google Play Store e Apple App Store), seguido de um trabalho contínuo de manutenção e melhorias com base no feedback dos usuários. Atualizações regulares garantem que o app permaneça compatível com novos dispositivos, versões de sistemas operacionais e evoluções nas APIs de terceiros, através de uma estrutura previamente configurada de CI/CD para implementações contínuas no código-fonte.
Esse processo estruturado permite desenvolver um aplicativo robusto e eficiente, que atenda às necessidades dos usuários, oferecendo uma experiência rica e integrada com dispositivos wearables e plataformas de saúde.
